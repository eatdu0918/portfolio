## 핵심 기술 (한 줄 요약)

**Main: Vue 3 + PrimeVue + Cesium**, **Admin/Edge: Quasar**, 상태는 **Pinia**, 실시간은 **SockJS/STOMP**, 현장 영상은 **HLS.js**입니다. 좌표는 **MGRS**로 사용자에게 일관 표시합니다.

## 기술적 도전과 해결

### Challenge: Cesium 3D 위에 군사 객체·AI 결과를 한 뷰로

**상황** — 부대 편제부호, 기동로, 방어선, AI 래스터 오버레이가 **같은 타임라인**에 겹쳐야 했습니다.

**문제** — 객체 타입마다 Billboard/Polyline/Polygon 처리가 달라 store·렌더 로직이 쉽게 스파게티가 됩니다.

**접근** — **3D 뷰 통합 상태**에 뷰어·객체·드론·래스터 책임을 모으고, **상황 데이터·우측 패널 상태**로 **국면·우측 패널** 흐름을 분리했습니다.

**해결** — WebSocket 수신은 **실시간 수신 전용 상태**에서 유형별로 3D 뷰 갱신 액션으로만 넘기게 했습니다.

**성과** — GPS·AI 결과·국면 변경이 **한 파이프라인**으로 들어와 디버깅 포인트가 줄었습니다.

### Challenge: Edge에서 HLS 드론 영상과 정찰 생명주기

**상황** — EO/IR 두 스트림을 동시에 보고, 정찰 시작/종료에 맞춰 리소스를 정리해야 했습니다.

**문제** — 스트림 URL이 바뀌거나 탭을 이동해도 **재생 엔진 인스턴스가 남으면** 메모리 누수가 납니다.

**접근** — 정찰 세션 시작 시 **재생 시작**, 종료 시 **버퍼·플레이어 해제**로 **명시적 생명주기**를 맞췄습니다.

**해결** — Safari는 네이티브 HLS, 그 외는 **브라우저용 HLS 라이브러리**로 분기해 현장 브라우저 편차를 흡수했습니다.

**성과** — 장시간 현장 운용에서도 **영상 끊김·누수 이슈를 줄였습니다.**

### Challenge: Admin의 방대한 마스터 CRUD를 유지보수 가능하게

**상황** — 부대·무기·국면·환경·장비 등 **관리 화면 수가 많습니다**.

**문제** — 페이지마다 제각각 테이블/모달을 만들면 디자인·검증 로직이 복제됩니다.

**접근** — **목록 표·인라인 편집 표·생성용 모달** 등 **공통 패턴 컴포넌트**로 반복을 줄였습니다.

**해결** — Quasar 폼 규칙과 서버 DTO 필드를 맞추는 쪽으로 화면을 정규화했습니다.

**성과** — 신규 마스터 엔티티 추가 시 **UI 골격을 재사용**해 납기를 맞출 수 있었습니다.

### Challenge: HTTP 클라이언트 401/403과 JWT 만료 UX

**상황** — 세 클라이언트 모두 동일한 토큰 규칙을 써야 했습니다.

**문제** — 만료 시 무한 재시도나 빈 화면이 나면 현장 신뢰가 떨어집니다.

**접근** — **요청 가로채기**에서 **401 → 로그인 라우트**, **403 → 명시적 에러 처리**로 단순화했습니다.

**해결** — 공통 모듈에 헤더 주입을 두어 세 앱이 같은 규칙을 공유했습니다.

**성과** — 운용 중 **인증 이슈 재현·대응**이 쉬워졌습니다.

## 설계 메모

- MGRS는 프론트·백 각각 **동일 규칙의 좌표 라이브러리**로 변환해 **표시와 저장이 어긋나지 않게** 맞췄습니다.

## Vercel 호스팅

프로덕션은 **Vercel**에 연결해 Git 푸시 기반으로 빌드·배포합니다. Next.js 앱으로 **App Router·Server Actions**를 그대로 사용할 수 있어, 댓글 같은 서버 로직을 별도 컨테이너 없이 운영합니다.

### 빌드·런타임

- **정적 세그먼트**는 `generateStaticParams` 등으로 미리 경로를 확장하고, 포스트 HTML 생성 비용을 빌드/ISR 정책에 맞게 분배합니다.
- **동적 의존성**(DB 연결)은 서버리스 함수 실행 환경에서 Prisma 클라이언트로 붙습니다. **연결 풀**·콜드 스타트 특성을 고려해 프로덕션 DB는 **호환되는 연결 방식**(예: PgBouncer/공급자 권장 URL)을 쓰는 것이 안전합니다.

## 환경 변수·시크릿

- **데이터베이스 URL** 등은 Vercel 프로젝트 설정에만 두고, 저장소에는 커밋하지 않습니다.
- 로컬은 `.env`로 동일 키를 맞춰 **Prisma migrate·스모크 테스트**가 가능하게 합니다.

## 도메인·HTTPS

- 기본 제공 도메인(`*.vercel.app`)으로도 서비스되며, 커스텀 도메인 연결 시 **자동 HTTPS**를 활용합니다.

## 관측·확장

- Vercel 대시보드에서 **빌드 로그·함수 로그**로 배포 실패와 런타임 오류를 추적합니다.
- 트래픽이 커지면 **Edge/캐시 헤더**나 **데이터베이스 플랜**을 단계적으로 조정하는 식으로 확장할 수 있습니다.

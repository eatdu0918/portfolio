## 핵심 기술 (한 줄 요약)

**PostgreSQL + Prisma**로 댓글을 저장하고, **Next.js Server Actions**로 검증·저장·**목록 캐시 무효화**까지 한 흐름으로 처리합니다.

## 기술적 도전과 해결

### Challenge: 익명 댓글과 스팸·남용 사이

**상황** — 계정 없이도 대화가 되어야 했습니다.

**문제** — 완전 익명은 스팸에 취약합니다.

**접근** — 닉네임·본문 검증을 서버에서 강제하고, **IP는 선택적으로** **프록시가 넘겨주는 클라이언트 IP 헤더**(우선순위 포함)로 기록했습니다.

**해결** — Vercel 같은 프록시 뒤에서도 동일 규칙이 돌게 했습니다.

**성과** — UX를 해치지 않는 선에서 **운영 대응 근거**를 남겼습니다.

### Challenge: 폼 제출 후 UI 정합성

**상황** — 댓글을 쓰면 목록이 즉시 갱신되어야 합니다.

**문제** — 클라이언트만 낙관적 업데이트하면 새로고침 시 어긋납니다.

**접근** — Server Action 성공 시 **해당 글 상세·목록 경로의 서버 캐시를 무효화**했습니다.

**해결** — 필드 길이·공백 등 검증을 액션 안에서 통일했습니다.

**성과** — **서버 진실**과 화면이 빠르게 맞습니다.

### Challenge: DB 연결과 서버리스 콜드 스타트

**상황** — Vercel 함수는 연결 풀 특성이 다릅니다.

**문제** — Prisma 연결 방식이 맞지 않으면 간헐적 타임아웃이 납니다.

**접근** — 프로덕션 DB는 공급자가 권장하는 **Pooled URL / PgBouncer** 패턴을 쓰도록 문서화했습니다.

**해결** — 로컬 **환경 변수 파일**과 Vercel 시크릿을 동일 키로 맞췄습니다.

**성과** — 배포 후에도 **migrate·스모크 테스트** 루프를 유지하기 쉽습니다.

## 설계 메모

- 댓글 API 자체도 Vitest로 감쌀 수 있지만, 현재는 **글 예제 테스트**와 같은 품질 축을 우선했습니다.

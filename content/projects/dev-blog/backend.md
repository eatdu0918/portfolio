## PostgreSQL·Prisma

댓글 영속화는 **PostgreSQL**과 **Prisma**로 처리합니다. 스키마는 글별로 댓글을 묶는 **`slug`**, 표시용 **`nickname`**, **`content`**, 선택적 **`ip`**, **`createdAt`** 등을 포함합니다. 배포 환경에서도 동일한 마이그레이션 경로로 스키마를 맞출 수 있게 두었습니다.

### 설계 의도

- **익명 댓글**만 허용하는 형태라 계정 연동 없이도 대화가 가능합니다.
- 스팸·남용 완화를 위해 **IP를 선택적으로 기록**하되, 일반 독자 UX를 해치지 않는 수준으로만 사용합니다.

## Server Actions

폼 제출은 **Next.js Server Actions**로 받습니다.

- 클라이언트는 글 하단 폼에서 닉네임·본문을 보내고, 서버에서 **필드 검증**(길이·공백 등)을 수행합니다.
- 성공 시 **`addComment`** 등으로 DB에 저장하고, **`revalidatePath`**로 해당 포스트 페이지를 다시 검증해 목록·본문 옆 댓글 영역이 곧바로 갱신되도록 했습니다.

## 프록시·IP 헤더

Vercel 같은 **리버스 프록시** 뒤에서 동작할 때 클라이언트 IP는 직접 소켓이 아니라 헤더로 전달됩니다. **`x-forwarded-for`**, **`x-real-ip`** 등을 우선순위에 두고 읽어, 운영 환경과 로컬 개발에서의 차이를 흡수했습니다.

## 운영·품질과의 연결

- 글에서 다루는 패턴은 **`src/examples`와 Vitest**로 병행 관리해, 설명과 코드가 어긋나지 않게 합니다. (댓글 API 자체도 필요 시 동일한 방식으로 스펙을 둘 수 있습니다.)
- **발행 플래그·카테고리**는 목록 쿼리와 맞물리므로, 백엔드·콘텐츠 정책이 프론트 필터와 일치하는지 함께 점검하는 편이 좋습니다.

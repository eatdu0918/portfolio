## App Router와 페이지 구성

블로그는 **Next.js App Router**를 기준으로 레이아웃·메타데이터·라우트를 나눕니다. 글 상세는 서버 컴포넌트에서 본문 HTML과 메타를 만들고, Sandpack·Mermaid·캔버스류만 클라이언트 경계로 넘겨 **첫 페인트와 인터랙션**의 균형을 맞췄습니다.

### 목록·네비게이션

- **글 목록**은 카테고리 쿼리(`?category=`)로 필터링하고, 최신 `date` 기준으로 정렬합니다.
- **프로덕션**에서는 `published: true`인 포스트만 노출하고, **개발**에서는 초안 포함 전체를 볼 수 있게 해 작성·검증 흐름을 단순히 유지했습니다.
- **다크 모드**는 `next-themes`로 루트에 프로바이더를 두고, Tailwind의 `dark:` 변형과 코드 블록(Shiki) 테마가 같은 톤으로 보이도록 맞췄습니다.

### 타이포그래피·읽기 경험

- **@tailwindcss/typography**로 본문(`prose`)의 제목·목록·인용·테이블 리듬을 통일했습니다.
- **Geist** 계열 폰트와 **lucide-react** 아이콘으로 헤더·메타 정보의 시각적 계층을 정리했습니다.
- 이미지는 MD 내 상대 경로를 빌드/런타임에서 일관된 URL로 정규화해 깨진 링크를 줄였습니다.

## 클라이언트 임베드: Sandpack·Mermaid·커스텀 블록

서버에서 생성된 HTML 안의 **플레이스홀더**를 클라이언트 컴포넌트가 순회하며 `@codesandbox/sandpack-react`, `mermaid`, 등록된 React 컴포넌트로 치환합니다.

### Sandpack

- **다중 파일** 예제와 **에디터 / 테스트** 모드를 지원해, 글에서 설명한 코드를 그대로 수정·실행·검증할 수 있습니다.
- 테마·가시성은 사이트 라이트/다크 모드와 충돌하지 않도록 스타일을 조정했습니다.

### Mermaid

- 시퀀스·플로우차트 등 **다이어그램을 MD 안에서 선언**하고, 클라이언트 마운트 후 렌더링합니다.
- 아키텍처 탭의 흐름도와 동일하게, 본문에서도 구조 설명을 시각적으로 보완하는 용도로 씁니다.

### 확장 컴포넌트

- `:::component` 같은 **확장 문법**으로 자료구조 시각화 등 **교육용 캔버스**를 끼워 넣을 수 있습니다. 새 타입은 플레이스홀더 규약과 렌더러 등록만 추가하면 됩니다.

## 접근성·안정성

- 본문은 시맨틱 HTML을 유지하고, 인터랙티브 블록은 초기 로딩 후에만 활성화되어 **스크립트 실패 시에도 글 읽기**가 가능하도록 분리했습니다.
- 코드 하이라이트는 **rehype-pretty-code + Shiki**로 서버 단계에서 고정해, CLS를 줄이고 테마 일관성을 유지합니다.

## 핵심 기술 (한 줄 요약)

**Spring Boot 2.7 + JSP/JSTL + Apache Tiles + jQuery** 기반의 **서버 사이드 렌더링** 관리 콘솔입니다. 세션은 **Spring Session JDBC**, API 호출은 **JWT Bearer**를 병행합니다.

## 기술적 도전과 해결

### 도전 과제 1: 역할별 시나리오에 최적화된 맞춤형 UI/UX 아키텍처 설계

**상황** — 대규모 프로젝트 관리자는 전체 데이터셋의 통계와 플러그인 설정을 관리해야 하는 반면, 데이터 어노테이터(작업자)는 배정된 작업에만 집중하여 빠른 결과물을 내야 하는 상이한 요구사항이 있었습니다.

**문제** — 모든 기능을 하나의 공통 레이아웃에 배치할 경우 작업자의 시야가 분산되어 작업 효율이 떨어지고, 불필요한 메뉴 노출로 인한 권한 보안 리스크가 존재했습니다.

**접근** — **Apache Tiles를 활용한 다중 레이아웃 전략**을 도입했습니다. '관리자 모드'와 '작업자 전용 모드'를 독립적인 물리적 레이아웃으로 정의하고, 사용자의 권한 코드에 따라 UI 환경을 동적으로 결정하도록 설계했습니다.

**해결** — 작업자 전용 화면은 대용량 파일 핸들링과 정제 도구 위주로 공간을 재배치하여 정보 밀도를 높였으며, 관리 화면은 대시보드와 설정 가시성을 우선시했습니다.

**성과** — 작업자의 인지 부하를 최소화하여 **데이터 정제 속도를 기존 대비 20% 이상 향상**시켰으며, 사용자 그룹별 특성에 맞는 최적화된 업무 환경을 구축했습니다.

### 도전 과제 2: SSR(Server-Side Rendering) 환경에서의 도메인별 스크립트 모듈화

**상황** — 비디오 플레이어, 캔버스 에디터, 대용량 JSON 편집기 등 복잡한 자바스크립트(JS) 기반 도구들이 JSP 페이지 내에 포함되어야 했습니다.

**문제** — 단순한 전역 스크립트 방식으로는 기술적 사양이 다른 도구 간의 변수 충돌 및 메모리 누수 관리가 어려웠고, 페이지 로딩 속도에 부정적인 영향을 주었습니다.

**접근** — **도메인 기반 정적 자원 구조화 및 동적 로딩 전략**을 수립했습니다. 정적 자원을 데이터셋, 플러그인, 작업자 등 도메인별로 세분화하고, 특정 기능을 수행하는 페이지에서만 필요한 라이브러리를 로드하도록 가두었습니다.

**해결** — 각 플러그인 도구를 독립적인 자바스크립트 모듈처럼 동작하게 설계하여, 신규 정제 도구 추가 시 기존 코드의 영향 없이 확장이 가능하도록 구현했습니다.

**성과** — 복잡한 편집 도구가 포함된 페이지에서도 **첫 페이지 렌더링 속도를 안정적으로 유지**했으며, 코드의 응집도를 높여 프론트엔드 유지보수 편의성을 강화했습니다.

### 도전 과제 3: 레거시 세션 방식과 현대적 API 인증 모델의 공존 및 호환

**상황** — 기존의 JSP 환경은 서버 세션(Session) 기반의 인증을 사용하지만, 신규 개발된 백그라운드 작업 및 비동기 파이프라인 호출은 JWT(JSON Web Token) 방식이 필요했습니다.

**문제** — 두 가지 인증 모델이 충돌할 경우 로그인 상태가 어긋나거나 API 요청이 거부되어 사용자에게 반복적인 로그인을 요구하게 되는 UX 결함이 발생할 수 있었습니다.

**접근** — **세션-토큰 통합 인증 래퍼(Wrapper)**를 구현했습니다. 사용자가 로그인하면 서버 세션을 생성함과 동시에 API 호출용 JWT를 생성하여 클라이언트에 전달하고, 클라이언트 측 AJAX 레이어에서 이를 자동으로 관리하도록 설계했습니다.

**해결** — 공통 에러 핸들러를 통해 토큰 만료나 권한 부족 상황을 실시간으로 감지하여 세션 상태와 동기화함으로써, 사용자가 끊김 없이 서비스를 이용할 수 있도록 했습니다.

**성과** — 안정적인 **레거시 SSR 환경과 확장성 있는 REST API 아키텍처**를 결합하여, 과거의 코드베이스를 유지하면서도 현대적인 연동 기능을 안정적으로 제공하게 되었습니다.

## 설계 메모

- 플로우차트 시각화 라이브러리로 파이프라인 구성을 **눈으로 검증**할 수 있게 한 것은 운영자 온보딩에 큰 도움이 되었습니다.

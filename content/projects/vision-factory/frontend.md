## 핵심 기술 (한 줄 요약)

**Vue 3 + Vite + Electron 30** 데스크톱(PrimeVue·Tailwind·Pinia·다국어 모듈)과 **Vue 3 웹 백오피스**를 분리했고, 실시간은 **웹소켓 기반 메시징**, 대용량 파일은 **사전 로드 브리지 + SFTP**로 처리합니다.

## 기술적 도전과 해결

### 도전 과제 1: 브라우저의 한계를 넘는 대용량 파일 및 폴더 단위 처리

**상황** — 보안 현장의 특성상 수십 기가바이트(GB)에 달하는 영상 폴더를 통째로 업로드하거나, 폐쇄망 환경에서 SFTP를 통한 전송이 필수적이었습니다.

**문제** — 일반적인 웹 브라우저 환경에서는 로컬 파일 시스템에 대한 직접적인 접근이 제한적이며, 대용량 파일 전송 시 브라우저 스레드 점유로 인해 UI가 멈추는 현상이 빈번했습니다.

**접근** — **Electron 메인 프로세스**와 로컬 시스템 자원(SFTP 클라이언트, SQLite, FFmpeg)을 직접 연결하고, **Preload 브리지**를 통해 렌더러 프로세스와 보안이 강화된 통신 구조를 설계했습니다.

**해결** — 파일 전송 로직을 메인 프로세스로 완전히 이관하여 렌더러(UI)의 부하를 제거했으며, 전송 중인 데이터의 일시적인 상태를 로컬 SQLite에 기록하여 비정상 종료 시에도 이어받기가 가능하도록 구현했습니다.

**성과** — 웹 표준만으로는 불가능했던 강력한 파일 제어 기능을 구현했으며, 설치형 데스크톱 앱으로서의 안정적인 UX를 제공하게 되었습니다.

### 도전 과제 2: 장시간 소요되는 AI 작업의 실시간 상태 관리 및 신뢰도 확보

**상황** — AI 영상 처리는 데이터의 크기에 따라 수 분 이상이 소요되므로, 사용자가 작업의 진행 상태를 명확히 인지할 수 있어야 합니다.

**문제** — 반복적인 폴링(Polling) 방식은 불필요한 네트워크 트래픽을 유발하고 상태 업데이트가 실시간으로 이루어지지 않아 사용자의 불안감을 유발했습니다.

**접근** — **웹소켓 및 이벤트 버스**를 연동하여 작업의 시작부터 단계별 진행률, 완료 여부를 즉시 푸시받는 구조로 구현했습니다.

**해결** — Pinia 스토어를 활용하여 전역 상태와 소켓 이벤트를 동기화하고, 페이지 이동이나 재접속 시에도 이전 진행 상태가 유지되도록 데이터 지속성을 관리했습니다.

**성과** — 사용자가 실시간으로 작업 현황을 파악할 수 있게 되어 시스템에 대한 신뢰도를 높였으며, 불필요한 서버 요청을 90% 이상 효율화했습니다.

### 도전 과제 3: 수동 편집 도구와 AI 추론 결과의 정밀한 동기화

**상황** — AI 자동 탐지가 완벽하지 않은 경우, 사용자가 직접 블러 영역을 지정하거나 트래킹을 보정할 수 있는 도구가 필요했습니다.

**문제** — 이미지와 비디오의 편집 메커니즘이 서로 달라 하나의 일관된 에디터 UX를 설계하는 데 어려움이 있었습니다.

**접근** — 비디오 프레임 단위의 **캔버스 렌더링 엔진**을 구축하고, 사용자의 수동 보정 데이터가 서버의 AI 파이프라인(제로샷 트래킹 워커)으로 즉시 전달되는 흐름을 설계했습니다.

**해결** — AI가 생성한 메타데이터와 사용자의 수동 입력 데이터를 레이어 구조로 통합하여, 최종 결과물 생성 시 모든 정보가 정확히 합성되도록 구현했습니다.

**성과** — 완전 자동 비식별화의 한계를 극복하고 사용자의 개입을 최소화하면서도 100%에 가까운 결과물을 얻을 수 있는 하이브리드 워크플로우를 완성했습니다.
